# 更新日誌

記錄 `screencast-trimmer` 每次版本的重要變更。

## [0.2.0] - 2026-05-30

### 新增
- 新增 `classify_filler.py`：規則式的逐字贅詞分類器，自動標記 Tier A + G2 + G3 應該剪掉的字幕段
- 涵蓋的規則：相鄰重複（`我覺得我覺得`）、單獨的 `OK`／`呢`、G2 贅詞（`就是`／`這個`／`一下`／`大概`／`這麼`）、`第N→第M` 自我糾正、起頭重複的口誤（`他的 X 他的`、`把 X 把`）
- 輸出 JSON 索引清單，可選擇同時輸出人類可讀的 `--review` SRT 供檢查
- 在 30 分鐘的 demo 影片上一次分類出 72 筆剪輯範圍（29 秒），全程一致，無需人工逐字判斷

## [0.1.0] - 2026-05-29

### 新增
- 首次發布：將冗長的螢幕錄影／demo 影片剪成精簡版本的工作流程
- 三層可疊加的剪輯：**空白靜音**（等待 AI 回應的死時間）、**口語贅詞**（語氣詞／口頭禪／重複／口誤）、**非必要語句**（重複論述、旁白、離題——由 LLM 做編輯判斷）
- 內附五支腳本：
  - `detect_silence_srt.py`：偵測靜音範圍輸出 SRT
  - `transcribe_words.py`：用 faster-whisper 產生**逐字**時間軸 SRT（中文為逐字／逐詞顆粒度）
  - `select_cues.py`：依索引挑選／剔除字幕段
  - `merge_srt.py`：依時間軸合併多個 SRT
  - `cut_silence.py`：依時間範圍剪掉影片片段（內建 VFR→CFR 修正，確保影音同步）
- 三項關鍵經驗寫入流程：
  - 螢幕錄影多為 VFR，直接剪會造成影音不同步——剪輯前強制轉 CFR，剪完必須核對影／音軌時長一致
  - 安靜的錄音需把靜音門檻設在 `mean_volume` 之下（預設 `-30dB` 會誤判語音為靜音）
  - 所有剪輯範圍合併成單一清單，**一次從原始檔剪出**，絕不在已剪過的檔案上疊加
