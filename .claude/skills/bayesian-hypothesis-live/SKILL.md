# 貝氏假說驗證工作流程（Live Demo 版）

## 工作流程總覽

```
設定框架 → 確立先驗 → 逐條更新（每步寫入 JSON）→ 歸納結論
```

每個階段都需要使用者確認後再推進。

---

## 第一階段：設定框架

### 1.1 理解原始假說

請使用者用一句話表達核心假說。識別假說的兩個維度：
- **主體**：誰／什麼正在被評估
- **判斷**：對主體的核心主張是什麼

### 1.2 設計競爭假說

原始假說不能單獨檢驗，必須有競爭對立面。提出 3–4 個互斥且窮盡的競爭假說，涵蓋可能的解釋空間。

**競爭假說設計原則：**
- 互斥：任意兩個假說不能同時為真
- 窮盡：所有假說加總應涵蓋全部可能性（機率和為 100%）
- 有張力：假說之間應有實質性的政策或行動含義差異

**常見競爭假說結構（可參考，不必照搬）：**

| 標籤 | 模板 |
|------|------|
| H₁ | 原始假說成立（核心變數 A 導致結果 B） |
| H₂ | 普遍現象，並非 A 特有的問題 |
| H₃ | 只在特定條件／場景下成立 |
| H₄ | 根源在於另一個變數 C，A 只是表象 |

詢問使用者是否需要調整，確認後再進入下一步。

### 1.3 設定先驗機率

先驗是在看任何證據之前，對每個假說的初始信念。

**提示使用者：**
- 若完全不確定，用均勻分布（每個假說等機率）
- 若已有背景知識，可以不均勻分布
- 所有假說機率之和必須等於 100%

記錄先驗，準備進入更新循環。

**確認先驗後，立即寫入初始狀態（step 0）：**

```json
{
  "hypotheses": ["H₁ <假說名稱>", "H₂ <假說名稱>", "H₃ <假說名稱>", "H₄ <假說名稱>"],
  "steps": [
    {
      "step": 0,
      "evidence": "先驗",
      "values": [
        { "prior": null, "posterior": <先驗值 0–1> },
        { "prior": null, "posterior": <先驗值 0–1> },
        { "prior": null, "posterior": <先驗值 0–1> },
        { "prior": null, "posterior": <先驗值 0–1> }
      ]
    }
  ]
}
```

`hypotheses` 陣列與 `values` 陣列按索引對應（`hypotheses[i]` 對應 `values[i]`）。假說名稱全程不變，只在頂層儲存一次。

使用 Write 工具將此 JSON 寫入 `bayes_state.json`（專案根目錄）。

## 第二階段：逐條證據更新

### 2.1 證據輸入格式

每條證據可以是：
- 論壇貼文／使用者留言
- 新聞報導
- 產品比較評測
- 資料點
- 專家觀點

### 2.2 每條證據的處理步驟

**步驟一：提煉核心資訊**

用 2–3 句話總結這條證據的關鍵事實，剔除雜訊。

**步驟二：估算概似度、執行貝氏更新，並輸出結果表格**

對每個假說估算條件機率（0–100%），完成貝氏歸一化計算，並以下列格式輸出：

| 假說 | 概似度 | 理由 |
|------|--------|------|
| H₁ | X% | ［一句理由］ |
| H₂ | X% | ［一句理由］ |
| ... | | |

更新後：H₁ X% · H₂ X% · H₃ X% · H₄ X%

**步驟三完成後，立即使用 Write 工具將完整歷史寫入 `bayes_state.json`：**

每次寫入時，保留所有先前的 steps，並在陣列末尾追加本輪新 step。

```json
{
  "hypotheses": ["H₁ <假說名稱>", "H₂ <假說名稱>", ...],
  "steps": [
    {
      "step": 0,
      "evidence": "先驗",
      "values": [...]
    },
    {
      "step": 1,
      "evidence": "<第一條證據摘要>",
      "values": [...]
    },
    {
      "step": <本輪序號>,
      "evidence": "<本條證據的一句話摘要>",
      "values": [
        { "prior": <本輪先驗 0–1>, "posterior": <本輪後驗 0–1> },
        { "prior": <本輪先驗 0–1>, "posterior": <本輪後驗 0–1> }
      ]
    }
  ]
}
```

數值使用小數（如 0.621），不使用百分比。每次覆寫整個檔案，但 `hypotheses` 不變，`steps` 陣列必須包含從 step 0 到本輪的所有記錄。

**步驟四：向使用者輸出訊號解讀（唯一對外可見的輸出）**

用一句話總結這條證據的方向性意義，格式：

> 這條證據**強力支持 ／ 輕微支持 ／ 未能區分** H₁，因為［一句理由］。H₂／H₃／H₄ 繼續被壓縮 ／ 略有回升。


### 2.3 滾動更新規則

- 每條證據的後驗，作為下一條證據的先驗
- 先驗從不重置，保持累積
- 可隨時暫停，總結當前狀態

### 2.4 階段性彙整格式

每 3–5 條證據後，輸出一張面向普通讀者的彙整表（不使用「先驗／後驗」術語）：

| 解釋 | 調查前 | 現在 | 原因 |
|------|--------|------|------|
| ［假說描述］ | X% | **Y%** ↑ | 支持此解釋的最強 2–3 條證據摘要 |
| ［假說描述］ | X% | Y% ↓ 基本排除 | 反驳逻辑：為什麼這條解釋站不住腳 |

**原因欄寫法：**
- 支持的假說：列出最強的 2–3 條關鍵證據，說明「為什麼相信」
- 被排除的假說：寫反駁邏輯，說明「為什麼不信」

## 第三階段：輔助分析工具

### 3.1 痛點／模式歸納

當證據來自使用者回饋或論壇時，在更新機率的同時，維護一張**模式歸納表**：

| # | 來源 | 模式類型 | 具體表現 |
|---|------|----------|---------|

隨著證據累積，將相似模式合併為 Pattern（A／B／C……），並追蹤：
- **踩坑者**：遇到這個問題的使用者
- **已突破者**：解決了這個問題的使用者

### 3.2 選擇性偏誤識別

主動識別證據來源的偏誤。常見偏誤：
- **論壇貼文**：只有遇到問題的人才發文，順暢使用者沉默
- **媒體報導**：負面事件更具新聞價值
- **評測文章**：作者往往有立場

在最終結論中註明偏誤可能造成的方向性影響。

### 3.3 證據品質分級

不同來源的證據權重不同，可在估算概似度時隱含考量：

| 品質等級 | 類型 | 說明 |
|---------|------|------|
| 高 | 官方發布、可驗證資料、專家實測 | 直接更新，權重大 |
| 中 | 媒體報導、使用者評論、比較文章 | 正常更新 |
| 低 | 匿名說法、單一資料點、行銷材料 | 保守更新，並註明不確定性 |

## 第四階段：結論輸出

### 4.1 最終後驗彙整

使用面向普通讀者的格式，不使用「先驗／後驗」術語：

| 解釋 | 調查前 | 現在 | 原因 |
|------|--------|------|------|
| ［假說描述］ | X% | **Y%** ↑ | 支持此解釋的最強 2–3 條證據摘要 |
| ［假說描述］ | X% | Y% ↓ 基本排除 | 反駁邏輯：為什麼這條解釋站不住腳 |

### 4.2 結論撰寫框架

**核心結論**（最高後驗假說）：
> ［假說內容］獲得 ［後驗］% 的支持。主要證據來自 ［最強的 2–3 條證據］。

**殘餘不確定性**（第二高假說）：
> ［假說內容］仍有 ［後驗］% 的可能性，原因是 ［尚未被充分排除的理由］。

**已排除的假說**：
> ［假說內容］已降至 ［接近 0 的後驗］%，被 ［關鍵證據］ 證偽。

**行動含義**：
> 基於以上分析，建議 ［具體行動］，但須注意 ［殘餘不確定性帶來的風險］。

### 4.3 選擇性偏誤修正聲明

若有顯著偏誤，在結論後附加：

> **偏誤說明**：本分析中 ［比例］% 的證據來自 ［偏誤來源］，可能導致 ［假說］ 的後驗被 ［高估／低估］。實際信心度區間建議在 ［範圍］ 內理解。

## 執行注意事項

### 概似度估算的常見陷阱

**陷阱一：概似度是條件機率，方向不能搞反**
- 錯誤：這篇貼文說明產品有問題，所以 H₁ 的概似度是 80%
- 正確：如果 H₁（系統性壁壘）為真，這種貼文出現的機率是多少？

**陷阱二：所有假說的概似度不需要加總為 100%**
- 每個假說的概似度是獨立估算的
- 加總的是【先驗 × 概似度】的乘積，再歸一化

**陷阱三：避免概似度趨同**
- 若所有假說的概似度估出相近數值，這條證據的區分度就為零
- 強證據應在假說之間產生明顯差異（如 40% vs 80%）

### 更新節奏建議

- 單條證據資訊量有限，不要過度解讀單條更新的結果
- 每 3–5 條證據後做一次階段性彙整
- 當某個假說後驗超過 70% 時，可開始討論是否還需要更多證據
- 當某個假說後驗跌破 5% 時，可標記為「基本排除」

### 何時可以停止收集證據

- 最高後驗假說穩定在 60%+ 且連續 2–3 條證據沒有顯著變動
- 或使用者明確表示已有足夠信心做決策
- 或證據來源已窮盡

## 快速參考：貝氏公式

```
P(H|E) = P(H) × P(E|H) / P(E)

其中：
P(H|E)  = 後驗：看到證據 E 後，H 為真的機率
P(H)    = 先驗：看到證據前，H 為真的機率
P(E|H)  = 概似度：如果 H 為真，證據 E 出現的機率
P(E)    = 歸一化常數 = Σ [P(Hᵢ) × P(E|Hᵢ)]
```

實作簡化版：

```
後驗（未歸一化）= 先驗 × 概似度
後驗（歸一化）  = 各假說未歸一化後驗 ÷ 所有假說未歸一化後驗之總和
```
